# Карточка модели классификации ролей

## Назначение

Определяет одно из направлений `ml_engineering`, `data_science`, `data_analysis`,
`data_engineering`, `python_backend`, `other` по описанию и явно указанным навыкам вакансии.

## Данные и признаки

Целевые классы создаются правилами из названия вакансии. Название исключено из признаков:
TF-IDF получает только очищенное `description` и `key_skills`, что предотвращает прямую
утечку target. Разбиение хронологическое при достаточной полноте `published_at`, иначе
стратифицированное со стабильным `random_state`.

## Алгоритм и метрики

Baseline — `DummyClassifier(most_frequent)`. Основная модель — TF-IDF по уни- и биграммам
и `LogisticRegression(class_weight="balanced")`. Основная метрика — macro F1; также
сохраняются weighted F1, accuracy, precision и recall по классам. Фактические результаты
появляются после обучения в `artifacts/role_classifier/*.json`.

## Ограничения и смещения

Слабые метки из заголовков могут ошибаться. Состав площадки, география, период сбора и
дисбаланс ролей создают смещения. Синтетические данные пригодны только для smoke-теста.
Модель не должна использоваться для решений о найме, ранжирования людей или оценки их
профессиональной пригодности.
